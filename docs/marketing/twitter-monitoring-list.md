## Engagement Triggers

### Respond Immediately

| Trigger | Action |
|---------|--------|
| Direct @mention with question | Answer or acknowledge |
| Support issue reported | Troubleshoot publicly, move to DM |
| Negative feedback | Acknowledge, offer to help |
| User sharing positive experience | Like, RT, thank |

### Respond Same Day

| Trigger | Action |
|---------|--------|
| Industry news in our space | Comment or QT with insight |
| Competitor mention (positive) | Monitor, no action |
| User question about EMS apps | Helpful answer (not sales-y) |
| Clinical question we can help with | Provide value, mention app if natural |

### Respond Within 48 Hours

| Trigger | Action |
|---------|--------|
| Industry thought piece | Engage in discussion |
| Research publication | Comment if relevant |
| Seasonal content opportunity | Create related content |
| User milestone (certification, etc.) | Congratulate |

## Weekly Monitoring Schedule

### Monday
- Full hashtag review
- Week's industry news scan
- Competitor activity check
- Plan week's proactive engagement

### Tuesday-Thursday
- Standard daily monitoring
- Real-time engagement
- Support response

### Friday
- Week's mention summary
- Top engagement opportunities
- Support ticket review
- Weekend content scheduling

### Weekend
- Reduced monitoring (2x/day check)
- Critical support only
- No proactive posting unless scheduled
