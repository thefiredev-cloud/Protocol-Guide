# Protocol Guide Twitter Content Calendar

Monthly planning template for Twitter/X content.

---

## Weekly Content Structure

| Day | Content Type | Purpose | Example |
|-----|--------------|---------|---------|
| **Monday** | Industry News / Week Kickoff | Position as informed, start conversations | "This week in EMS..." or news commentary |
| **Tuesday** | Educational / Protocol Tip | Provide value, establish expertise | Clinical pearl, protocol reminder |
| **Wednesday** | Feature Spotlight | Product awareness | Feature highlight, user tip |
| **Thursday** | Community / User Content | Build relationships | User shoutout, question for audience |
| **Friday** | Lighter Content / Engagement | End week on high note | EMS humor (appropriate), weekend wishes |
| **Weekend** | Reduced / Support Only | Rest, monitor only | Only post if major news or scheduled |

## Real-Time Slots (Reserve)

Buffer 50% of posting capacity for reactive content:

| Slot | Purpose | Example |
|------|---------|---------|
| Breaking News | Industry news response | AHA guideline update |
| User Interaction | Community engagement | Responding to viral EMS post |
| Support Public | Public support response | Acknowledging widespread issue |
| Trend Participation | Relevant trend | EMS-related trending topic |

---

## Content Approval Workflow

1. **Draft** - Content creator writes tweet + visual
2. **Review** - Second team member reviews for accuracy and tone
3. **Medical Check** (if clinical) - Licensed provider reviews
4. **Schedule** - Add to scheduler with visual
5. **Monitor** - Watch for engagement after posting

---

## Visual Asset Checklist

For each week, prepare:

- [ ] 1-2 app screenshots (features, search results)
- [ ] 1 infographic or protocol visual (if educational post)
- [ ] Header image updated if needed
- [ ] Video/GIF if feature demo week

---

## Monthly Review Questions

At end of each month, answer:

1. **Top performing content type**: Which pillar got most engagement?
2. **Top performing tweet**: What made it work?
3. **Missed opportunities**: What could we have engaged with but didn't?
4. **Feedback received**: What are users asking for?
5. **Next month focus**: What should we do more/less of?
